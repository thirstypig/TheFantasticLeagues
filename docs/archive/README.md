# docs/archive/

Historical documents kept for the record, **not** surfaced in the in-app
`/docs` viewer (the viewer's glob deliberately excludes this folder).

## What lives here

- **`plans/`** — feature/fix/refactor plan docs whose work has **shipped**.
  They explain *why* code is the way it is; kept for the git-visible trail.
  Archived, not deleted, per the doc-management policy (2026-07-10).
- **`brainstorms/`, `scratch/`** — exploratory notes, superseded.

## What is NOT archived

- **Records** (audit reports under `docs/reports/`, solution post-mortems
  under `docs/solutions/`, `FEEDBACK.md`) stay in place — they are
  institutional memory, never archived or deleted.
- **Live plans** stay in `docs/plans/` (e.g. `standings-nested-stats-refactor.md`
  while that refactor is in progress) and binding decisions
  (`*-direction-lock.md`).

## Policy (2026-07-10)

Distinguish **living** docs (edit in place, must stay current) from
**records** (dated, append-only, never edited/deleted) from **superseded
plans** (archive here once shipped). Prefer archiving over hard-deletion so
history — and the reasoning behind shipped work — is never lost.
